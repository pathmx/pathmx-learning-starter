# Learning Pattern Catalog

Prefer ordinary Markdown unless a richer installed pattern materially improves
the learning move.

| Pattern | Use when | Source |
| --- | --- | --- |
| Prose Block | A concise explanation, contrast, or worked example is enough | Built in |
| Question | The learner should provide durable evidence | Built in |
| Accordion | Optional detail should stay available without interrupting the main move | Native `<details>` |
| Flashcard | Retrieval should happen before the answer is revealed | [Learning components](../assets/learning.components.md) |

The coach may name only entries listed here. An unavailable optional pattern
falls back to readable prose or a built-in Question.
